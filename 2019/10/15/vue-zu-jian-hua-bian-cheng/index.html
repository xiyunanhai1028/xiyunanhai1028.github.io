1. Vue扩展插件

  • vue-cli：vue脚手架
  • vue-resource(axios)：ajax请求
  • v-router：路由
  • vuex：状态管理
  • vue-lazyload：图片懒加载
  • vue-scroller：页面滑动关系
  • mint-ui：基于vue的UI组件库（移动端）
  • element-ui：基于vue的UI组件库（PC端）

2. 使用vue-cli创建模板项目

2.1 说明：

vue-cli是vue官方提供的脚手架工具

2.2 创建vue项目

  • 老版本
    • npm install -g vue-cli
    • vue init webpack 项目名
    • cd 项目名
    • npm install
    • npm run dev
    • 访问：http://localhost:8080/
  • 3.x版本
    • npm install -g @vue/cli or yarn global add @vue/cli
    • vue create 项目名 or vue ui
    • cd 项目名
    • npm install
    • npm run dev
    • 访问：http://localhost:8080/

2.3 模板项目结构

|-- build : webpack 相关的配置文件夹(基本不需要修改) 

​    |-- dev-server.js : 通过 express 启动后台服务器 

|-- config: webpack 相关的配置文件夹(基本不需要修改) 

​    |-- index.js: 指定的后台服务的端口号和静态资源文件夹 

|-- node_modules 

|-- src : 源码文件夹 

​    |-- components: vue 组件及其相关资源文件夹 

​    |-- App.vue: 应用根主组件 

​    |-- main.js: 应用入口 js 

|-- static: 静态资源文件夹 

|-- .babelrc: babel 的配置文件 

|-- .eslintignore: eslint 检查忽略的配置 

|-- .eslintrc.js: eslint 检查的配置 

|-- .gitignore: git 版本管制忽略的配置 

|-- index.html: 主页面文件 

|-- package.json: 应用包配置文件 

|-- README.md: 应用描述说明的 readme 文件

3. 项目打包与发布

3.1 打包

npm run build

3.2 发布-使用静态服务器工具包

  • npm install -g serve
  • serve dist

3.3 发布-使用动态web服务器（tomcat）

  • 修改配置：webpack.prod.conf.js
output:{
    publicPath:'/XXX/'   //打包文件名称
}
  • 重新打包
npm run build
  • 修改dist文件夹为项目名称：xxx
  • 将xxx拷贝到运行的tomact的webapps目录下

4. eslint

4.1 说明

    1. ESLint是一个代码规范检查工具
    2. 它定义了很多特定的规则，一旦你的代码违背了某一规则，eslint会做出非常有用的提示
    3. 官网：http://eslint.org/
    4. 基本以替代以前的JSLint

4.2 ESLint提供以下支持

  • ES
  • JSX
  • style检查
  • 自定义错误和提示

4.3 ESLint提供以下几种校验

  • 语法错误校验
  • 不重要或丢失的标点符号，如分号
  • 没法运行到的代码块（使用过webStorm的应该了解）
  • 未被使用的参数提醒
  • 确保样式统一规则，如sass或者less
  • 检查变量的命名

4.4 规则的错误等级有三种

  • 0：关闭规则
  • 1：打开规则，并且作为一个警告（信息打印黄色字体）
  • 2：打开规则，并且作为一个错误（信息打印红色字体）

4.5 相关配置文件

  • .eslintrc.js:全局规则配置文件
'rules':{
    'no-new':1
}
  • 在js/vue文件中修改局部规则
/*eslint-disable no-new*/
new Vue({
    el:'body',
    components:{app}
})
  • .eslintignore:指令检查忽略的文件
*.js
*.vue

6. 组件间通信

6.1 组件间通信基本原则

    1. 不要在子组件中直接修改父组件的状态数据
    2. 数据在哪，更新数据的行为（函数）就应该定义在哪

6.2 vue组件间通信方法

  • props
  • vue的自定义事件
  • 消息订阅与发布（如：pubsub库）
  • slot
  • vuex

6.3 组件间通信-props

6.3.1 使用组件标签时
<my-component :name='张萨姆' :age='3' :set-name='setName'></my-component>
6.3.2 定义MyComponent时

    1. 在组件内声明所有的props

    2. 方法一：只指定名称

      props:['name','age','setName']

    3. 方法二：指定名称和类型

      props:{
    name:String,
    age:Number,
    setName:Function
}

    4. 方法三：指定名称/类型/必要性/默认值

      props:{
    name:{
        type:String,
        required:true,
        default:'李四'
    }
}
6.3.3 注意
    1. 此方法用于父组件向子组件传递数据
    2. 所有标签属性都会成为组件对象的属性，模板页面可以直接引用
    3. 问题：
      1. 如果需要向非子后代传递数据必须多层逐层传递
      2. 兄弟组件间也不能直接props通信，必须借助父组件才可以
6.3.4 案例
  • 父组件
<template>
  <div>
    <p>我是父组件</p>
    <p>我是子组件</p>
    <Childe :name='name' :age='18' :set-name='setName' />
  </div>
</template>

<script>
import Childe from './childe'
export default {
  data() {
    return {
      name: '我是从父组件传递过来的',
      age:18
    }
  },
  components: {
      Childe
  },
  methods: {
    setName(){
      alert('父类方法')
    }
  }
}
</script>

<style>
</style>
  • 子组件
<template>
  <div>
    <p>{{name}}</p>
    <p>{{age}}</p>
    <button @click='setName'>点击按钮</button>
  </div>
</template>

<script>
export default {
  props: {
    name:String,
    age:Number,
    setName:Function
  }
}
</script>

<style>
</style>

6.4 组件间通信-vue自定义事件

6.4.1 绑定事件监听

    1. 方法一：

      <my-component @delete_todo='deleteTodo'></my-component>

    2. 方法二：

      this.$refs.xxx.$on('delete_todo',function(todo){
    this.deleteTodo(todo)
})
6.4.2 触发事件
this.$emit(eventName,data)
6.4.3 注意
    1. 此方法只用于子组件向父组件发送消息（数据）
    2. 问题：隔代组件或兄弟组件间通信此方法不合适
6.4.4 案例
<script>
        //全局组件
        Vue.component('Htext', {
            template:`
                <div>
                    <h3>{{msg}}</h3>
                    <button @click='changeColor'>变色</button>
                </div>
            `,
            data() {
                return {
                    msg:'我是全局组件'
                }
            },
            methods: {
                changeColor(){
                    this.$emit('change')
                }
            },
        })
        //局部组件
        let App={
            template:`
                <div>
                    <h3>{{msg}}</h3>
                    <Htext @change='changeColor'></Htext>
                </div>
            `,
            data() {
                return {
                    msg:'父组件',
                    colors:['blue','green','red'],
                    c:'blue',
                    num:0
                }
            },
            methods: {
                changeColor(){
                    console.log('--------')
                }
            },
        }
        new Vue({
            el:'#app',
            template:`<App/>`,//传递
            components:{
                App
            }
        })
    </script>

6.5 组件间通信-消息订阅与发布（PubSubJS库）

6.5.1 引入库
npm install --save pubsub-js
6.5.2 订阅消息
PubSub.subscribe('msg',function(msg,data))
6.5.3 发布消息
PubSub.publish('msg',data)
6.5.4 注意
  • 有点：此方法可以实现任意关系组件间通信（数据）
6.5.5 事件的2个重要操作
    1. 绑定事件监听（订阅消息）
      1. 目标：标签元素
      2. 事件名（类型）：click/focus
      3. 回调函数：function(event){}
    2. 触发事件（发布消息）
      1. DOM事件：用户在浏览器上对应的界面上做对应的操作
      2. 自定义：编码手动触发

6.6 组件间通信-slot

6.6.1 理解

此方法用于父组件向子组件传递标签数据

6.6.2 子组件：Child.vue
<template>
  <div>
    <slot name="xxx">不确定的标签结构1</slot>
    <div>组件确定结构</div>
    <slot name="yyy">不确定的标签结构2</slot>
  </div>
</template>
6.6.3 父组件：Parent.vue
<child>
    <div slot="xxx">xxx对应的标签结构</div>
    <div slot="yyy">yyy对应的标签结构</div>
</child>
