1. Lambda

1.1 基本语法：

Java8中引入了一个新的操作符-> 该操作符称为箭头操作符或Lambda操作符

箭头操作符将Lambda表达式拆分成两部分：左侧：Lambda表达式的参数列表；右侧：Lambda表达式中所需执行的功能，即Lambda体

1.2 语法格式：

  • 语法格式一：无参数，无返回值

    () -> System.out.println(“Lambda!”)

@Test
public void test1() {
    Runnable r = () -> System.out.println("Lambda!");
    r.run();
}

  • 语法格式二：有一个参数，无返回值

    (x) -> System.out.println(“Lambda!”)

  • 语法格式三：如只有一个参数，小括号可以省

    x -> System.out.println(“Lambda!”)

  • 语法格式四：有两个及以上的参数，有返回值

    Comparator c = (x, y) -> x + y;

    @Test
public void test2() {
    Comparator<Integer> c = (x, y) -> x + y;
    int compare = c.compare(1, 2);
    System.out.println(compare);
}

  • 语法格式五：有两个及以上的参数，有返回值，且有多条语句

    Comparator c = (x, y) -> {

        System.out.println("Lambda");
    return x + y;
};
    @Test
public void test3() {
    Comparator<Integer> c = (x, y) -> {
        System.out.println("Lambda");
        return x + y;
    };
    int compare = c.compare(1, 2);
    System.out.println(compare);
}

public List<Person> predicateTest(List<Person> p, Predicate<String> predicate) {

1.4 方法引用：

如Lambda体中的内容有方法已经实现了，我们可以使用方法引用(可以理解为方法引用是Lambda表达式的另一种表现形式)

主要有三种格式：

  • 对象：：实例方法名
  • 类：：静态方法名
  • 类：：实例方法名

==注意：==

  • Lambda体中调用方法的参数列表与返回值类型，要与函数式接口中抽象方法的函数列表与返回值类型保持一致
  • 若Lambda参数列表中的第一个参数是实例方法的调用者，而第二个参数是实例方法的参数时，可以使用ClassName：：method
案例一：对象：：实例方法名
@Test
public void test1(){
    Person person = new Person();
    //没有使用方法引用
    Consumer<String> c=(x)->person.setName(x);
    c.accept("张三");
    System.out.println(person.getName());//张三

    //使用方法引用
    Consumer<String> c1=person::setName;
    c1.accept("李四");
    System.out.println(person.getName());//李四
}

通过上面的例子，我们可以看出我们可以把Consumer c=(x)->person.setName(x);简化为 Consumer c1=person::setName;通过对象：：实例方法名称，省去了不少代码

方法引用我们需要注意的是

@FunctionalInterface
public interface Consumer<T> {
    void accept(T var1);
}

这个接口的方法accept(T var1)是一个参数，且无返回值

我们再来看看person的setName方法

public void setName(String name) {
    this.name = name;
}

同样是一个参数，且无返回值

案例二：类：：静态方法名
@Test
public void test2(){
    Function<Integer,String> fn2=String::valueOf;
    String apply1 = fn2.apply(12);
    System.out.println(apply1);//12
}

我们来看一下valueOf方法的代码

public static String valueOf(@RecentlyNullable Object obj) {
    throw new RuntimeException("Stub!");
}

通过代码我们知道valueOf方法是一个参数，且带返回值的静态方法

再来看看apply方法

R apply(T var1);

同样是一个参数，且带返回值

案例三：类：：实例方法名
@Test
public void test3() {
    BiPredicate<String, String> bp = (x, y) -> x.equals(y);
    boolean test = bp.test("1", "2");
    System.out.println(test);

    //使用方法引用
    BiPredicate<String, String> bp1 = String::equals;
    boolean test1 = bp1.test("1", "2");
    System.out.println(test1);
}

上面实例中方式列表中第一个参数x,是实例方法的调用者，第二个参数是y是实例方法的参数

我们再来看一下equals方法的代码

public boolean equals(@RecentlyNullable Object anObject) {
    throw new RuntimeException("Stub!");
}

他既不是静态方法，调用它时也不是实例对象调用

  • 格式：ClassName:：new

2. Stream流

2.1. 什么是Steam?
  • 是数据渠道，用于操作数据源（集合，数组等）所生成的元素序列
  • 集合讲的是数据，流讲的是计算

==注意：==

①Stream自己不会存储元素

②Stream不会改变源对象。相反，他们会返回一个特有结果的新Stream

③Stream操作是延迟执行的。这意味着他们会等到需要结果的时候才执行

2.2. Stream三步操作：
    1. 创建Stream
    2. 中间操作
    3. 终止操作
2.3. 创建Stream有四种方式
//1，通过集合创建Stream
List<String> list=new ArrayList<>();
Stream<String> stream = list.stream();

//2，通过Arrays中的静态方法
Person[] p=new Person[10];
Stream<Person> personStream = Arrays.stream(p);

//3，通过Stream中的静态方法of
Stream<String> stringStream = Stream.of("张三", "李四");

//4，创建无限流
//迭代
Stream<Integer> iterate = Stream.iterate(0, (x) -> x + 2);
//生成
Stream<Double> generate = Stream.generate(() -> Math.random());
2.4 Stream中间操作

多个中间操作可以连接起来形成一个流水线，除非流水线上触发终止操作，否则中间操作不会执行热河的处理。而在终止操作时一次性全部处理，称为“惰性求值”

筛选与切片：

方法 描述

disinct() 筛选，通过流所生成元素的hashCode()和equals()去除重复元素
limit(long maxSize) 截断流，使其元素不超过给定数量
skip(long n) 跳过元素，返回一个扔掉了前n个元素的流。若流中元素不足n个，则返回一个空流。与limit(n)互补
